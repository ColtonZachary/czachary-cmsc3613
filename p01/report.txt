1. Which version of the list implementations is better: contiguous or linked?

    The linked list is better for this project because

    It allows efficient insertions anywhere without shifting elements O(1) if position is known.

    Memory is allocated dynamically, so it does not have a fixed maximum size like the array-based list (max_list).

    The contiguous (array) list is simpler and can be slightly faster for traversal because elements are stored 
    sequentially in memory, which is cache-friendly.

    Trade-off: For frequent sorted insertions and unknown list sizes, linked lists are more flexible. 
    For small, fixed-size lists, contiguous arrays are slightly faster.

2. How to understand the function void traverse(void (*visit)(List_entry &))?

    traverse iterates through every element in the list and applies a function pointer visit to each element.

    For example, if visit is print_record, it will print all records in the list.

    This design decouples list storage from operations on elements, making the list reusable for different 
    tasks (printing, processing, modifying).

3. What’s the search algorithm you used in Task 3? Why is it suitable?

    Algorithm: Linear search

        Loop through the list from start to end.

        For ID lookup, check if record.ID == searchID.

        For name prefix lookup, check if first_name and last_name start with the given prefix.

    Why suitable:

        The list is sorted, but the search is for prefix matches or exact matches, not a full ordering.

        Linear search guarantees that all matches are found without relying on more complex structures.

        It works correctly for both linked and array implementations.

4. Can you directly use comparison operators (>, <, >=, etc.) on Personal_record instances?

    No, because C++ does not know how to compare two Personal_record structs automatically.

    Solution used: Implemented a comparison function "comes_before".

    This function defines ordering based on last name → first name → ID.

    We use it to maintain sorted order in the list and during insertion.
